title: H-DenseUNet论文速览（一）

H-DenseUNet论文速览

一、背景

肝癌是世界上最常见的癌症疾病之一，每年导致大量死亡。CT的准确测量，包括肿瘤体积，形状，位置和进一步的功能性肝体积，可以帮助医生做出准确的肝细胞癌评估和治疗计划。传统上，肝脏和肝脏病变由放射科医生逐层描绘，这很耗时，并且容易出现评分者和评分者之间的差异。因此，临床上对肝肿瘤的自动分割方法要求很高。

由于肝脏与其他邻近器官之间的低强度对比度，从造影剂增强的CT体积自动分割肝脏是一项非常具有挑战性的任务。此外，放射科医生通常通过注射方案来增强CT扫描，以便清楚地观察肿瘤，这可能会增加肝脏区域图像内的噪声。与肝脏分割相比，肝肿瘤分割被认为是一项更具挑战性的任务。首先，肝肿瘤在一个患者体内具有各种大小、形状、位置和数量，这阻碍了自动分割，如图1所示。其次，一些病变没有明确的边界，限制了仅基于边缘的分割方法的性能（参见图1第三行中的病变）。第三，许多CT扫描由各向异性尺寸组成，沿z轴方向变化很大（体素间距范围从0.45mm到6.0mm），这进一步给自动分割方法带来了挑战。

为了解决这些困难，人们提出了许多分割方法，包括强度阈值、区域生长和可变形模型。然而，这些方法依赖于手工制作的特征，并且特征表示能力有限。最近，全卷积神经网络（FCN）在广泛的识别问题上取得了巨大成功。许多研究人员在肝脏和肿瘤分割问题中使用深度学习方法推进了这一进程，文献可大致分为两类。

(1) 2D FCN，例如如UNet架构、多通道FCN和基于VGG-16的FCN

(2) 3D FCN，其中 2D 卷积被具有体积数据输入的 3D 卷积取代

二、现有方法缺点

基于 2D FCN 的方法忽略了 z 轴上的上下文，这将导致分割精度有限。具体来说，从体积图像中裁剪出的单个或三个相邻切片被输入到 2D FCN 并且通过简单地堆叠 2D 分割图来生成 3D 分割体积。尽管采用了相邻切片，但仍不足以探测沿三维的空间信息，这可能会降低分割性能。为了解决这个问题，一些研究人员提出使用三平面方案或 RNN 来探测 3D 上下文 。例如，Prasoon 等人。 在正交平面（例如，xy、yz 和 xz 平面）上应用了三个 2D FCN，体素预测结果由这些概率的平均值生成。与 2D FCN 相比，3D FCN 的计算成本和 GPU 内存消耗较高。高内存消耗限制了网络的深度以及滤波器的视野，这是性能提升的两个关键因素。 3D 卷积的繁重计算也阻碍了其在训练大规模数据集方面的应用。此外，许多研究人员已经证明了知识转移（从一个源领域学到的知识有效地转移到另一个领域）对于提高性能的有效性。不幸的是，缺乏 3D 预训练模型，这限制了 3D FCN 的性能和采用。

三、H-DenseUnet特点简介

  1. 网络架构

    首先，为了充分提取高级切片内特征，我们根据 2D 卷积的预定义设计原则设计了一个非常深且高效的网络，称为 2D DenseUNet，其中同时具有密集连接路径 和 UNet 连接的优点融合在一起。密集连接路径源自密集连接网络（DenseNet），其改进的信息流和参数效率减轻了深度网络的训练难度。与DenseNet 不同，我们在架构中的编码部分和解码部分之间添加了UNet连接，即长程跳跃连接；因此，该网络可以实现低级空间特征保留，以实现更好的切片内上下文探索。

  2. 混合特征探索

    其次，为了探索体积特征表示，我们设计了一个名为 H-DenseUNet 的端到端训练系统，其中有效提取片内和片间特征，然后通过混合特征融合（HFF）层联合优化。具体来说，3D DenseUNet 通过自动上下文机制与 2D DenseUNet 集成，这是堆叠通用性的一般形式 。在 2D DenseUNet 语义概率的指导下，可以很好地减轻 3D DenseUNet 中的优化负担，这有助于提高 3D 上下文提取的训练效率。此外，通过端到端系统，由体积特征和高级代表性切片内特征组成的混合特征可以自动融合并联合优化，以更好地识别肝脏和肿瘤。

四、核心方法

我们采用级联学习策略来减少整体的计算时间，该策略在许多识别任务中也得到了应用。首先训练一个简单的Res Net架构，得到快速但粗略的肝脏分割。在感兴趣区域( region of interest，ROI )中，我们提出的H - Dense UNet通过2D Dense UNet f2d和3D Dense UNet f3d有效地探测层内和层间特征，然后在混合特征融合( hybrid feature fusion，HFF )层中联合优化混合特征，以实现精确的肝脏和病灶分割。

层与层之间的密集连接的一个优点是它比传统网络具有更少的输出维度，避免了学习冗余的特征。此外，密集连接的路径保证了层与层之间的最大信息流，改善了梯度流，从而减轻了在一个深度网络中搜索最优解的负担

然而，原始的DenseNet - 161 是针对物体分类任务设计的，而我们的问题属于分割主题。此外，一个用于分割任务的深度FCN网络实际上包含多个最大池化和上采样操作，这可能会导致低级(即高分辨率)特征的信息丢失。基于以上两点考虑，我们开发了一个2D DenseUNet，它继承了密集连接路径和类UNet连接的优点。具体来说，在每个微块内部使用层与层之间的密集连接以确保最大的信息流，而UNet长距离连接将编码部分和解码部分链接以保留低层信息
